The documentation for this repository is partially generated at publish time by the powershell script specified in [sample-generator](./sample-generator.md). We should move to a model where all documentation is treated as a Github pages artifact, linked from the root-level readme.md. [ci.yaml](/.github/workflows/ci.yml) should build the documentation and upload it i.e. using `actions/upload-pages-artifact` and `actions/deploy-pages`. Existing code to build the CLI reference from the central catalog should be refactored to fit this model. [cli-schema.md](../cli-schema.md) should be built as part of this process based on the [raw schema](../schemas/bgraster-config.schema.json). Any content that's currently in the md and can't be automatically derived should be added to the schema to make a similar output format possible.
